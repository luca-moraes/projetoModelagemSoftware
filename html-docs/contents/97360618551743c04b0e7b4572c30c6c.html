Login

UMLUseCase
Untitled :: Model :: Login

Description

Fluxo principal:

  1. Usuário informa as credenciais de acesso.

  2. Caso as informações estejam corretas, o login é liberado e data e hora do login anterior é exibida.

Fluxo alternativo:

  1. As informações estão erradas e o login é negado.

  2. Caso o login seja negado 'n' vezes, o acesso daquele usuário é bloqueado temporariamente e o include caso de uso "Aviso de bloqueio" é ativado.

Caso de uso referentes aos requisitos :

  • 2, 3 e 46.
